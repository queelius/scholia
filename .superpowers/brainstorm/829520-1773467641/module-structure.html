Module Structure After v0.3.0

How the codebase looks after all three features land. New modules in green, modified in yellow, unchanged in gray.

Python Modules — texwatch/
__init__.py — version
config.py — .texwatch.yaml loading
workspace.py — project discovery
watcher.py — watchdog integration
compiler.py — subprocess execution, log parsing
synctex.py — SyncTeX parsing
structure.py — sections, TODOs, word count
bibliography.py — bib parsing, citation cross-ref
environments.py — LaTeX environment parsing
digest.py — document metadata
changes.py — section-level diffs
persistence.py — SQLite compile history (NEW)
awareness.py — user focus tracking, highlights, annotations, screenshots (NEW)
labels.py — \label parsing + /labels endpoint logic (NEW)
server.py — HTTP/WS routes (SLIMMED — delegates to persistence, awareness)
mcp_server.py — MCP tools (ENRICHED — user_focus, highlights, annotations)
cli.py — lightweight CLI (SLIMMED — no feature parity with MCP)
JavaScript Modules — texwatch/static/
viewer.js — PDF.js wrapper, SyncTeX UI
filetree.js — file tree widget
dashboard.js — multi-project dashboard
autocomplete.js — 4 completion providers (commands, envs, citations, refs) (NEW)
snippets.js — snippet definitions + expansion (NEW)
awareness.js — focus reporting, highlight/annotation rendering (NEW)
editor.js — CodeMirror setup (MODIFIED — loads autocomplete, snippets, awareness)
